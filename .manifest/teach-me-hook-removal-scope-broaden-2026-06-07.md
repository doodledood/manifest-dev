# Definition: teach-me skill — drop broken Stop hook, broaden to "any topic", sync distributions

## 1. Intent & Context
- **Goal:** Apply the PR #181 review outcomes to the `teach-me` skill: remove the non-functional prompt-type Stop hook, broaden scope from "current session's work" to "any topic" (a PR, an ADR, or whatever the user names), rebalance the description, sharpen the sole-enforcement prose — then propagate to all distribution targets and ship.
- **Mental Model:** `teach-me` is a behavior-activator skill in the `manifest-dev-tools` plugin. Its value is the relentless teach-until-demonstrated loop. The Stop hook was meant to enforce that but is non-functional: prompt-type hooks get only the hook input JSON (a `transcript_path`, not the conversation) and cannot read files, so the gate decided blind. Dropping it makes the body prose the sole enforcement. The skill is distributed to codex/pi/opencode under `dist/`, and `.claude/skills/teach-me` is a symlink to the `claude-plugins/manifest-dev-tools` source — edit the source.

## 2. Approach
- **Architecture:** Single-source edit to `claude-plugins/manifest-dev-tools/skills/teach-me/SKILL.md`; version bumps; regenerate `dist/` via `sync-tools`; lint; ship.
- **Execution Order:**
  - D1 (skill edits) → D2 (version bumps) → D3 (sync distributions) → D4 (project gates: README + lint) → D5 (ship: commit/push/PR)
  - Rationale: source and versions must settle before sync regenerates `dist/` from them; gates run on the final tree; ship last.
- **Risk Areas:**
  - [R-1] Description exceeds 1024-char frontmatter limit after broadening | Detect: char count of the `description` value.
  - [R-2] `sync-tools` leaves `dist/` partially regenerated or `component-namespaces.json` missing teach-me | Detect: teach-me present and current in all three dist trees + namespace metadata.
  - [R-3] Pi version drift between `package.json` and `pi-cli.md` example | Detect: both strings equal.
- **Trade-offs:**
  - [T-1] Hard enforcement vs simplicity → Prefer simplicity (drop the hook), per user decision; re-add later if early-exits prove a real problem.
