# Definition: figure-out terminal contract + --team mode restructure

## 1. Intent & Context
- **Goal:** Fix figure-out's intermittent "goes ahead and implements" failure by restoring a strengthened universal terminal contract, and stop figure-out-team drifting from figure-out by converting it to a `--team` override mode of figure-out with a thin discovery wrapper.
- **Mental Model:** Commit `321a692` deleted figure-out's terminal contract ("the user calls for any proposal or edit… point to `/define`") while sharpening the convergence gate; the surviving restraint line is scope-qualified to "clarifying answers," so solution agreement reads as authorization and the harness's action bias fills the vacuum. Separately, figure-out-team is a stale copy of an older figure-out core (no belief register, rival set, probe files, terminal contract). The fix: one terminal contract in figure-out (consumed three ways: interactive offers `/define`; `--autonomous` surfaces the read and stops; `--team` posts wrap-up to Slack and offers `/define` to the operator in chat), and a `--team` flag loading `references/team.md` overrides (precedent: `references/autonomous.md`) so team mode inherits all future figure-out upgrades.
- **Source surface:** edit `claude-plugins/manifest-dev/skills/` only — `.claude/` resolves via symlinks; `dist/` is generated by sync-tools.

## 2. Approach
*Initial direction, not rigid plan.*

- **Architecture:** D1 rewrites figure-out's restraint paragraph into the universal terminal contract. D2 adds the `--team` flag + `references/team.md` carrying all Slack mechanics from today's figure-out-team body, moves `slack-mrkdwn.md` under figure-out's references, folds the team log rule into team.md. D3 shrinks figure-out-team's SKILL.md to a directive invocation of figure-out with `--team`. D4 is housekeeping + dist regeneration + PR.
- **Execution Order:** D1 → D2 → D3 → D4. Rationale: D2 references the contract D1 lands; D3 depends on D2's flag existing; D4 ships the result.
- **Risk Areas:**
  - [R-1] sync-tools regeneration touches unrelated dist files | Detect: review `git diff --stat` scope after running it.
  - [R-2] Thin wrapper fails to activate figure-out with `--team` in dist harnesses (codex/opencode/pi) | Detect: inspect generated dist wrapper; cross-skill directive invocation is the repo's endorsed pattern.
  - [R-3] figure-out SKILL.md balance degrades (accretion) | Detect: review-prompt gate (AC-1.2).
- **Trade-offs:**
  - [T-1] Sync-by-architecture vs self-contained copy → Prefer `--team` flag + override file because drift already caused a real failure once; override-file conflicts are managed by the established autonomous.md "what changes / what stays" pattern.
  - [T-2] Static prompt review vs behavioral evals → Prefer static review gates now; no eval harness exists for "agent stops at the read" (see ASM-1).
