# Definition: figure-out trustworthiness redesign — evidence-ledger trust core, thin mode refs, DIAGNOSIS/RESEARCH task files, dist sync

## 1. Intent & Context

- **Goal:** Implement the figure-out trustworthiness redesign agreed in the 2026-06-11 figure-out session and recorded in ADRs `20260611-figure-out-spine-owns-epistemics-mode-refs-thin`, `20260611-figure-out-evidence-ledger-and-independent-rederivation`, `20260611-figure-out-task-taxonomy-diagnosis-research-topic-trigger` (read them before implementing — they are the authoritative design record). Land the change on branch `claude/figure-out-skill-trustworthy-x56fba` / PR #189.
- **Mental Model:** figure-out produces *process trust* — the Read (conclusion + confidence + Evidence Ledger + overturn conditions) is the deliverable. All epistemics live in the spine (`SKILL.md`), mode-general; mode references (`LOG.md`, `autonomous.md`) carry only their mode's mechanics. Task files carry only non-natural probing fuel. figure-out is standalone: its methodology never depends on other tools existing; the only cross-tool pointers are the `/define` offer and the degrade-gracefully prompt-engineering hook. Source surface is `claude-plugins/manifest-dev/skills/figure-out/` (`.claude/` is symlinked to it); `dist/` is generated by sync-tools.

## 2. Approach

*Initial direction, not rigid plan. Expect adjustment when reality diverges.*

- **Architecture:** Rewrite the spine to absorb the trust core via replace-before-add (fold into existing sentences where the ADRs say so: discriminating probes into the rival-set line; status decay into the claim-discipline tier). Thin the two mode references to pure deltas. Add two task files, slim BUG.md, sweep all six for orthogonality against the new spine and each other. Then metadata/docs, then regenerate dist, then drive PR #189 to mergeable.
- **Execution Order:**
  - D1 (spine) → D2 (mode refs) → D3 (task files) → D4 (versions + READMEs) → D5 (dist sync) → D6 (PR landed)
  - Rationale: spine wording is the reference point for thinning the refs (no restatement) and sweeping task files (no spine duplication); dist regenerates from finished sources; lifecycle gate last.
- **Risk Areas:**
  - [R-1] Spine absorbs six mechanisms and becomes a checklist | Detect: review-prompt MEDIUM+ findings on density/anti-patterns; spine reads as steps rather than principles.
  - [R-2] Thinned refs accidentally drop genuinely mode-specific content (e.g. autonomous ASM-flagging, log path resolution) | Detect: diff review against the pre-change files; review-prompt restructuring check.
  - [R-3] dist targets drift (Pi excludes runtime-owned skills; OpenCode/Codex flavor differences) | Detect: AC-5.1 verifier compares dist content against source.
- **Trade-offs:**
  - [T-1] Spine completeness vs leanness → Prefer leanness at equal coverage: each mechanism is one principle-shaped clause, not a procedure ("status decay" is a sentence, not a section).
  - [T-2] Task-file coverage vs signal → Prefer fewer, genuinely non-natural probes; a probe the model raises unprompted dilutes the file (best-upfront = orthogonality sweep, not length).

### Deliverable 1: Rewritten figure-out spine (`claude-plugins/manifest-dev/skills/figure-out/SKILL.md`)

**Acceptance Criteria:**

- [AC-1.1] The spine carries the mode-general trust core, each element principle-shaped: (a) tiered claim discipline — epistemic status always voiced (verified/inferred/assumed), concrete provenance required for load-bearing claims, forming the Evidence Ledger; (b) read anatomy — the named read ships conclusion, confidence, its Evidence Ledger, and what would overturn it (trade-off boundary for judgment reads), collapsing for evidence-free calls; (c) discriminating probes folded into the rival-set sentence (split rivals / survive best disconfirming test, not support accumulation); (d) status decay — verified expires when its basis does, decayed pillars re-anchor at read time; (e) independent re-derivation at convergence — fresh context that hasn't seen the conclusion gets question + ledger evidence, fires when the read is load-bearing and no human will audit it or on request, cheap version (no new evidence gathering), skip-with-disclosure where no isolation exists; (f) earned non-convergence — "underdetermined" only when every runnable discriminating probe has been run and is in the ledger.
  ```yaml
  verify:
    prompt: |
  ```
