name: review
description: 'Autonomous PR review that posts high-signal, human-voiced comments under your account. Spawns a tiered reviewer fleet (always-on core + content-gated specialists), filters to medium+ findings, runs a single holistic coherence pass grounding findings against PR history, bundle context, and the author''s manifest, then submits a single GitHub PR review (decision `comment`). --loop bypasses approval, watches the PR with backoff, per-comment verifier judges addressed-by-fix / addressed-by-valid-reply / needs-pushback (one round of pushback per thread), reruns on success, terminates at 3 cycles or 24h. Use when reviewing someone else''s PR — or your own manifest-driven PR — and you want a precision-tuned review without babysitting. Triggers: review pr, autonomous review, post pr review, autoreview, loop review, watch pr review.'
argument-hint: '[pr-url] [--manifest <path>] [--bundle <urls>] [--loop]'
user-invocable: true
---

**Resolve inputs.** Take `pr-url` from the arg or infer from the current branch's upstream PR; no PR → abort with a one-line reason. If `--manifest <path>` is given, use it as intent ground; otherwise auto-discover from `.manifest/` only when the PR's head branch belongs to you (skip for someone else's PR — their conventions aren't yours). Parse the PR description and conversation for linked PRs (`Depends on #N`, `Stack: #A,#B`, `Co-changes: org/repo#N`, GitHub PR URLs); merge with `--bundle <urls>` (comma-separated); cap at 5. Cross-repo links skip silently when MCP access is unavailable — note the skip in the plan.

**Run the fleet.** Spawn reviewer agents in parallel. **Always-on core:** `change-intent-reviewer`, `code-bugs-reviewer`, `code-design-reviewer`, `code-maintainability-reviewer`, `code-simplicity-reviewer`, `context-file-adherence-reviewer`. **Content-gated additions** (route by what the diff touches): `type-safety-reviewer` on typed-language files (`.ts/.tsx/.py/.java/.kt/.go/.rs/.cs`); `test-quality-reviewer` and `code-testability-reviewer` on source code (skip on docs/config-only diffs); `contracts-reviewer` on API surfaces (HTTP clients, routes, RPC stubs, SDK calls); `operational-readiness-reviewer` on CI/infra/env/migrations/workers/queues/secrets; `docs-reviewer` and `prose-value-reviewer` on prose (`.md`, comments, docstrings); `prompt-reviewer` on prompts/skills/agents (and `prompt-token-efficiency-verifier`, `prompt-compression-verifier` when those gates apply). Forward the manifest to every spawned agent when present. **Reviewer agents stay narrow-lens — never pass them PR conversation, linked-PR diffs, or linked-PR conversation.** That context belongs to the holistic pass and verifier only. Filter the aggregated findings to Medium+ (drop Low).

**Coherence and post.** Spawn one holistic-pass subagent using `references/HOLISTIC_PASS.md` — it sees the Medium+ findings plus PR history, bundle context, and manifest; prunes, dedupes, decides anchor per finding (inline/file/PR-level), rewrites in human voice using `references/VOICE.md`, and optionally drafts a summary header. Present the resulting review plan via ExitPlanMode showing comments-to-post (with anchors and bodies), what context was loaded, and any truncation. On approval, submit a **single GitHub PR review** with decision `comment` (never `request_changes`, never `approve` here) — all comments batched atomically. **Zero Medium+ findings after the coherence pass:** prompt the user `"Looks good to me — post as approval on the PR?"` via AskUserQuestion. Approve → submit a GitHub review with **decision `approve`** and body `Looks good to me.`. Decline → exit silent, no PR action. No AI disclosure footer is ever added to any comment, summary, or reply — there is no flag for it.

**`--loop`.** Bypass approval entirely — initial post and every rerun run autonomously. After posting, `subscribe_pr_activity` and wait with backoff per round: 15 → 30 → 60 → 120 minutes; events resolve the wait early. On each wake, for every pending comment spawn a per-comment verifier subagent using `references/VERIFIER.md` (it gets the original finding, the author's reply on the thread, the post-fix code at the anchor if a later commit modified it, the full PR history, and bundle context). Dispositions: `addressed-by-fix` and `addressed-by-valid-reply` mark the comment resolved; `needs-our-pushback` posts the verifier's drafted reply as a **thread reply on the existing comment** (not a new review) — **one round of pushback per thread maximum, then drop**. All comments resolved → rerun the full pipeline on the new PR state (the holistic pass dedupes against our own prior comments via PR history); fresh backoff. **Termini, first to fire wins:** (1) clean rerun + zero new findings + user approves the lgtm prompt → submit `approve` review, exit; (2) lgtm declined → exit silent; (3) 3 cycles total (initial post + 2 reruns); (4) 24h wall-clock backstop; (5) the 4-tier wait cap elapses with comments still pending. On (3)/(4)/(5): silent unsubscribe + chat summary (cycles completed, comments still pending). Never post a bump comment on the PR.
